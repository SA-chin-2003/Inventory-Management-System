## ğŸ“‚ Code Structure
The program follows a standard procedural approach:

| Routine | Description |
| :--- | :--- |
| **Data Declaration** | Defined custom types (`ty_inventory`) and global tables (`gt_inventory`). |
| **Selection Screen** | User input parameter (`P_RECS`) to define how many records to process. |
| **GET_DATA** | Populates the internal table with simulation data. |
| **PROCESS_DATA** | Loops through the table to apply business calculations and status logic. |
| **BUILD_FIELDCAT** | Manually configures columns (e.g., enabling "Summation" for Total Value). |
| **DISPLAY_ALV** | Renders the final output to the user. |
